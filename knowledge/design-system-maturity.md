# Design System Maturity Model

## Levels

Level 1: Ad-hoc
- Raw values, duplicated styles, inconsistent components.

Level 2: Shared Components
- Reusable components exist, but tokens, variants, and governance are incomplete.

Level 3: Tokenized
- Color, type, spacing, radius, elevation, state, and motion tokens are used consistently.

Level 4: Governed
- Contribution rules, accessibility gates, component ownership, versioning, and review workflows exist.

Level 5: Enterprise Platform
- Multi-brand, multi-mode, cross-platform design system with analytics, documentation, compliance support, and mature governance.

## Reporting Rule
Every major XD audit or implementation should report current maturity, target maturity, blockers, and next actions.
