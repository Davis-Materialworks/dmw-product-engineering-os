# Tesler's Law

## Principle
Every system has irreducible complexity; decide whether the product or user should bear it.

## Cite When
Cite when workflows push complexity onto users instead of absorbing it with better defaults or automation.

## XD Critique Pattern
- Identify the affected user decision or action.
- Explain the likely cognitive, motor, or trust cost.
- Recommend a design change that preserves intent while reducing friction.
