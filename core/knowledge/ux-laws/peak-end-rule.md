# Peak-End Rule

## Principle
Users judge experiences disproportionately by emotional peaks and endings.

## Cite When
Cite when completion, error, recovery, checkout, onboarding, or support flows end weakly.

## XD Critique Pattern
- Identify the affected user decision or action.
- Explain the likely cognitive, motor, or trust cost.
- Recommend a design change that preserves intent while reducing friction.
